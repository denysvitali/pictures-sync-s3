===================================================================================
EXAMPLES OF EXPLOITABLE BUGS
===================================================================================

1. COMMAND INJECTION (Critical)
   --------------------------------
   Current code ACCEPTS this:

   settings.SetRemote("remote; rm -rf /", "/photos")
   settings.SetRemote("remote\n--delete-excluded", "/path")

   Impact: If remote name is used in shell command, arbitrary code execution!

2. RACE CONDITION (Critical)
   --------------------------------
   Save() uses RLock instead of Lock:

   // 100 goroutines all call SetRemote() simultaneously
   // All call Save() which uses RLock
   // All write to settings.json.tmp at the same time
   // File corruption guaranteed!

   Proof: TestConcurrentSettingsUpdates shows 1000+ errors

3. VALIDATION BYPASS (Critical)
   --------------------------------
   Current code:

   s := settings.DefaultSettings()
   s.RemoteName = "malicious; drop table users--"  // Direct field access!
   s.Save()  // Saves malicious value to disk

   Impact: All setter validation can be bypassed

4. INVALID VALUES ACCEPTED (High)
   --------------------------------
   Current code ACCEPTS:

   settings.SetReformatThreshold(-100.0)     // Negative
   settings.SetReformatThreshold(math.NaN()) // Not a number
   settings.SetReformatThreshold(math.Inf(1)) // Infinity
   settings.SetTransfers(-50)                 // Negative transfers
   settings.SetCheckers(9999999)              // Huge value (resource exhaustion)

   Impact: System crashes, unexpected behavior, resource exhaustion

5. CANNOT SET ZERO VALUES (Medium)
   --------------------------------
   User tries:

   settings.SetReformatThreshold(0.0)
   settings.Save()
   // Load settings again...
   settings.Load()
   fmt.Println(settings.GetReformatThreshold())  // Output: 0.3 (not 0!)

   Impact: User configuration ignored, confusing behavior
